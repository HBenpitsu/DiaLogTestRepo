
# MUST
- 日記機能:
    - 対話機能: ユーザはAIとの対話を通じて日記の内容を書き出す.
    - 形成機能: 対話形式の内容を日記として見返しやすい形で整理する.　（この際, 対話記録と形成結果は分けて保存しておく）
    - 閲覧機能: 過去の日記を閲覧することができる.
- エクスポート機能: 記述した日記の内容などをmd形式でエクスポートできる.
- セルフホスト機能: ユーザのデバイスのファイルシステム上に日記を保存できる.
- BYOK機能: ユーザが自身のgeminiのAPIキーを登録して使用できる.

# WANT
- パーソナライズ機能: ユーザの属性や好み（ペルソナ：明示的に指定）や過去の対話内容（会話コンテキスト：自動抽出）を反映したAI応答を行う。これらを統合してユーザプロファイルとして管理する。
- 日記機能:
    - 修正機能: ユーザは直接生成された日記の内容を編集する事ができる.
- トピック機能: トピック付きの複数の日記を同じ日に複数作成できる
- 天気・ニュース候補提示機能: 日記の新規作成（対話開始）時に、その日の天気情報や主要なニュースの候補を自動的に取得し、ユーザが選択したものを日記に紐付けて記録できる.
    - 天気API: https://developers.google.com/maps/documentation/weather/overview?hl=ja
    - ニュースAPI: https://gnews.io/
    - デザインへの反映: 提示された候補からユーザが採用・棄却を選択し、採用されたものを日記エントリの表示に紐付ける.


- 月間/週間まとめ機能: その内容についても自分で修正できると良い.
    - 生成機能: AIが一定期間の日記の内容を整理してまとめる.
    - 修正機能: AIが生成したまとめを修正することができる.
        - 修正時, 対象期間の日記(月間まとめの生成時には週間まとめ)を閲覧しながら修正を行う事ができる.
    - 閲覧機能: 生成されたまとめを閲覧できる. (修正ボタンで修正機能を呼び出せると良い)
- ニュース機能: その日にあったニュースを関連付けられる
    - 取得機能: ニュースの取得は https://gnews.io/ を使えそう.
    - 選択機能: 取得した候補の中から、日記に残したいものをユーザが選択する.
    - 感想はユーザが日記のエントリに書く

# MAYBE
- 分量表示機能: 対話中, 日記としてどれくらいの分量になるのかの目安を常に確認できる
- PDFエクスポート機能: PDF形式でエクスポートできる

- 統計機能: 頻出ワードなど日記から統計を取れる
- トレンド機能:
    - 統計処理: (ユーザの同意を得て) ユーザ全体の頻出ワードや, 注目を得ているニュースを算出する
    - 結果閲覧: ユーザは処理の結果を取得して確認できる.
- タグ機能: 複数の日記エントリをグルーピングできる
- 音声入力機能
- 音声出力
- マスコット機能: 対話中, マスコットが話しているかのようなUIとする (live 2Dなど?)

# FUTURE

- 位置追跡機能: その日の移動記録を日記に紐付けられる

- 感情推定機能: 日記から感情を推定して記録できる.
