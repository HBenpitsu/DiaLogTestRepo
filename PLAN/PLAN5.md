# Implementation Plan 5: Cloud Readiness & Data Portability

## 1. 概要
マルチテナントに対応し、クラウド環境でも利用可能にする。また、クラウド版からセルフホスト版へのスムーズな移行機能（エクスポート）を実現する。

## 2. 実現範囲 (Scope)
- **認証**: Google SSO (OIDC) 連携。
- **クラウドストレージ**: Firestore 等を用いたデータ永続化。
- **BYOK (Cloud)**: クッキーを用いた API キーの安全な保持。
- **エクスポート/移行**: 指定期間の日記・まとめを、セルフホスト用ディレクトリ構造で一括ダウンロード（ZIP アーカイブ）。

---

## 3. 具体的な実装手順

### Phase 1: 認証とマルチテナント
1. **Auth Middleware**: Google ID Token の検証と、リクエストコンテキストへの `userId` 注入。
2. **Firestore Adapter**: ユーザー ID ごとに隔離されたデータアクセスの実装。
3. **設定の切替**: 環境変数 `APP_MODE` に基づくアダプターの動的な切り替えロジック。

### Phase 2: BYOK (Bring Your Own Key) の正式対応
1. **セキュア・クッキー管理**: API キーをバックエンドに保存せず、ブラウザのクッキーとして保持。
2. **設定 UI**: クッキーへの書き込みと削除のインターフェース。

### Phase 3: エクスポートと移行
1. **マイグレーション・パッケージ生成**: `docserver-arch.md` の仕様に従い、Markdown ファイル群とメタデータを ZIP 形式でアーカイブ。
2. **データポータビリティの確認**: ダウンロードしたファイルをセルフホスト版のストレージルートに展開して正常に動作することを確認。

---

## 4. マイルストーン
- [ ] **Step 1**: Google ログインとクラウド環境でのデータ保存。
- [ ] **Step 2**: ユーザ個別の API キー設定（BYOK）の完了。
- [ ] **Step 3**: エクスポートによるセルフホスト移行の実現。
