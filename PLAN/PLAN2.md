# Implementation Plan 2: Robust Infrastructure & Search

## 1. 概要
PLAN1 で作成した MVP の「保存と閲覧」を強化し、実用的なパフォーマンスと利便性を確保する。インメモリのインデックスを SQLite3 に置き換え、外部エディタでの編集を許容するファイル監視機能を導入する。

## 2. 実現範囲 (Scope)
- **インデックス管理**: SQLite3 を導入し、Markdown の Frontmatter を高速検索可能にする。
- **検索機能**: 日付範囲、トピック、タグなどによる絞り込み機能。
- **ファイル監視**: ローカルの Markdown ファイルが外部ツール（Obsidian 等）で変更された際に、自動的にインデックスを更新する。
- **Document Server ロジックの洗練**: バックエンド内のリポジトリ層を整理し、将来的な独立プロセス化を見据えた設計にする。

---

## 3. 具体的な実装手順

### Phase 1: SQLite3 インデックスの導入
1. **スキーマ定義**: `documents`, `document_tags` 等のテーブル作成（`docserver-arch.md` 準拠）。
2. **インデックス構築ロジック**: 起動時および明示的な要求時にディレクトリを再スキャンして DB を更新する機能。
3. **リポジトリの移行**: `LocalDiaryRepository` が SQLite3 を参照して一覧・検索を行うように変更。

### Phase 2: 検索 API と UI
1. **検索 API の追加**: `ListDiaries` にフィルタ条件（日付範囲、キーワード）を追加。
2. **フロントエンド検索バー**: 日記一覧画面にフィルタリング UI を追加。

### Phase 3: ファイル監視 (Watcher)
1. **fsnotify 等の導入**: Go の `fsnotify` ライブラリを使用してディレクトリを監視。
2. **同期ロジック**: ファイルの追加・変更・削除を検知し、SQLite3 インデックスを差分更新する処理の実装。
