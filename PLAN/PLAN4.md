# Implementation Plan 4: Aggregation & Automation

## 1. 概要
日々の記録を積み重ね、週間・月間の「まとめ」を自動またはオンデマンドで生成する。セルフホスト環境でのタスクスケジューリングを実現する。

## 2. 実現範囲 (Scope)
- **要約生成ロジック**: 特定期間の日記を AI で要約する `SummaryService` の実装。
- **要約の永続化**: 週間・月間まとめを Markdown ファイルとして保存（`summaries/` ディレクトリ）。
- **要約閲覧・編集 UI**: まとめ画面のリスト・詳細・編集機能。
- **バックグラウンドスケジューラ**: 定期的な自動生成（セルフホスト時）。

---

## 3. 具体的な実装手順

### Phase 1: 要約の生成と保存
1. **要約プロンプトの設計**: 複数の日記をバランスよくまとめるためのプロンプト。
2. **要約リポジトリの実装**: `ISummaryRepository`（Markdown 保存と SQLite3 インデックス）。
3. **要約一覧・詳細画面**: 週間・月間の切り替え表示と Markdown レンダリング。

### Phase 2: 自動化とスケジューリング
1. **インメモリ・スケジューラの導入**: `robfig/cron` 等を用いた軽量なジョブ管理。
2. **自動実行トリガー**: 設定された周期（毎週月曜等）に要約タスクを実行。
3. **状態管理**: 生成済みの要約がある場合は重複生成を避けるロジック。

---

- [ ] **Step 1**: 期間を指定した「まとめ」の手動生成の完了。
- [ ] **Step 2**: 週間・月間まとめの UI 表示と編集機能。
- [ ] **Step 3**: バックグラウンドでの自動生成機能の安定化。
