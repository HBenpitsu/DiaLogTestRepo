# Implementation Plan 3: Rich Context & Personalization

## 1. 概要
AI との対話をより豊かにするため、外部情報（天気・ニュース）を対話のコンテキストに含め、ユーザのプロファイルを考慮した応答を実現する。

## 2. 実現範囲 (Scope)
- **外部 API 連携**: `IWeatherProvider` および `INewsProvider` の実装（初期は Built-in キー前提）。
- **候補選択 UI**: 対話開始前に天気・ニュースの候補を表示し、ユーザが採用・棄却を選択。
- **ユーザプロファイル**: ユーザが明示的に設定する「ペルソナ」を対話に反映。
- **自動コンテキスト抽出**: 対話内容から、AI が重要な出来事や好みを抽出し、長期記憶として管理。

---

## 3. 具体的な実装手順

### Phase 1: 外部情報の取得と提示
1. **Provider 実装**: OpenWeatherMap や GNews 等の外部 API 連携。
2. **候補提示 UI (UIServer)**: 対話開始画面に天気・ニュースのカードを表示し、選択状態を管理。
3. **対話への統合**: 選択された情報を AI へのシステムプロンプトに注入。

### Phase 2: パーソナライズ (Persona & Context)
1. **設定画面の実装**: ユーザプロファイル（ペルソナ）の編集 UI。
2. **会話コンテキスト抽出**: 対話終了時に AI に要約を依頼し、`UserProfile` に保存。
3. **継続的な学習**: 次回の対話開始時に、過去の会話コンテキストを読み込み、AI が過去の話を覚えているような振る舞いを実現。

---

## 4. マイルストーン
- [ ] **Step 1**: 今日の天気・ニュースを考慮した対話の実現。
- [ ] **Step 2**: ユーザの好みに合わせた応答のパーソナライズ。
- [ ] **Step 3**: 過去の対話内容を次回の対話に活かす仕組みの完了。
